MPI
The Algorithm
  • Node{id: int; color: int; nodes: List<Node>}
  • Before running any computations check if the graph can be colored with the available colors: if the count of neighbors >= the number of available colors, then the graph cannot be colored
  • For each node in the list of nodes delegate a task to check the colors of its neighbors. Hold a list of all the colors and remove the color of each neighbor of the node. Choose the first valid color, assign it to the node and continue with the next task 
The Performance measurements
  • 5 Nodes - 80ms
  • 25 Nodes - 191ms
  • 100 Nodes - 1167ms
  • 250 Nodes - 6606ms
  • 500 Nodes - 36862ms

Multi-threaded 
The Algorithm
  • Every thread finds the right color for a node, by checking every neighbour what color it has and generates a random color different from neighbour’s color.
  • Used mutex to sync the work when coloring a node.
